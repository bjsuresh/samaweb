Document Management

SAMA Document Manager provides complete solution for all your Plant documentation needs- Engineered Documents such as P&ID or Process Flow drawings, Data sheets & specifications, Instrument index, point data base, hook up or installation drawings, power supply & distribution, loop drawings; equipment or system manuals; procedures and standards. It helps in maintaining these documents in an highly encrypted form with powerful user access control and version control. Proposer user can add new documents or propose changes with a new version. Approver user approves and it becomes available to assigned users for view.
Static documents and dynamic data Static documents are those documents like P&IDs, layout drawings, process flow drawings, etc.
● These can be edited with the editors provided and keywords attributed, so that they be searched for keywords (CAD, pdf, word excel documents can be searched without keywords also)
● They are related to tag numbers, equipment, units or areas or plants during configuration or later
● They can be part of major document categories such as standards, engineered documents, manuals, procedures etc”. each these can further be sub categorized- eg engineered documents can be sub divided into P&IDs, piping layout, ec.
Dynamic Data Dynamic data typically includes parameters such as instrument ranges, alarm & trip sets, tuning constants; and properties such as IO allocation details which includes cable termination, junction box details, data sheet related details like operating temperature, pressure etc. All such details are directly stored in relational tables the database. User can query for Dynamic Data and view them, make changes, propose and approve.
Query All these can be queried, displayed and printed based on queries on tag, equipment, unit, area hierarchy AND/OR document category AND/OR document name AND OR any text.

Alarm Management

SAMA Alarm Management suite of software is part of Supra Advanced Manufacturing Analytics (SAMA). SAMA AS Basic package, Alerts, Analysis, Metrics CC TV integration. Web analytics. Enterprise server. Master Alarm Database. Dynamic Alarming. Change Management, secure OPC Client. Rationalisation including priority assist. And operator assist. Root Cause Analysis module.
Basic Package – Data Logger with AE client Captures data from serial or Ethernet ports - typically data from DCS SCADA PLC SOE SNMP Traps, Postscript data, Files, ftp, OPC A&E, OPC DA, ODBC or direct connectivity with database such as Structware . Alarm server handles multiple date-time formats, event definition – alarms, operator actions, system diagnostic messages; track configuration changes and other messages; detect and store log reports, format & archive data. Detect alarm floods. Feature Rich Client, unique set of windows for each user, powerful real time search, sorting and filters. Alerts can be sent based on Logical processing of events. Alerts can be made via Email, SMS, Dial out, audible alerts by speakers, PLC DO, Modbus, CCTV. CCTV camera can automatically be adjusted to its preset so that upon occurrence of a particular alarm, live video of equipment in alarm can be played to operator. Users can be alerted in groups as well as based on Shift Rota. Escalation based on time-in-alarm as well as frequency can be configured. If an alarm stays for more than say 5 minutes or f it alarms more than a certain number of times in a time window, it can be escalated. With OPC DA interface, alarms can be suppressed or equipment can be shut down or started, based on logical combination of alarms.
Analysis with dynamic charts as required by EEUMA 191 for ISA 18.2 or Priority Distribution, Standing alarms, stale alarms, unacknowledged alarms, frequency, time-in-alarm & between alarms, alarm sequence display, chattering alarms and symptomatic alarms. Comply with ISA 18.2 standards. View reports for plant, area, unit, equipment over different time scales. Auto store, print and e mail reports. Web based advanced alarm analytics for multiple users caters to multiple users- management, operations, specialists across different sites based on access levels.

Rationalisation

Rationalisation module provides the user with a simple and efficient tool along with secure OPC client.
To ensure consistency in alarm & trip point settings for similar equipment
To ensure correct priority is calculated for alarms
To define Operator assist - Alarm description, consequences, action to be taken and time to respond.
To reduce configured alarms to reduce uncontrollable alarm load
To correct mis-configured settings, to correct alarm setting for nuisance / chattering alarms, to remove Duplicate alarms.
To aid in State based, multi process mode alarm systems
Audit & enforce
Rationalization process: Query- export to XL - rationalization meeting- import XL - follow change management procedures.
